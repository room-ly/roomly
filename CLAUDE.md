# Roomly — プロジェクト横断ルール

## 概要

賃貸管理会社向けSaaS。物件・入居者・契約・家賃・修繕・オーナー送金を一元管理する。

## プロジェクト構成

```
estate/
├── roomly/           ← メインアプリ（Next.js App Router + TypeScript）
├── hp/               ← ホームページ（Next.js）
├── lp/               ← ランディングページ（Vite + React + TS）
├── supabase/         ← 共有バックエンド（DB / Auth / Storage）
│   └── migrations/
├── images/           ← ロゴ・素材
└── 事業運営/          ← 事業運営タスク管理
```

## 共有バックエンド（Supabase）

全アプリが同一のSupabase DBを参照する。スキーマは `supabase/migrations/` で管理。

### RLS（Row Level Security）

全テーブルに `company_id` カラムがあり、JWT内の `company_id` クレームでフィルタする。
`public.company_id()` 関数でJWTからテナントIDを取得。

## 機能スコープ（MVP）

### 物件系
- 物件管理（建物・部屋の登録、間取り・設備・写真）
- 空室管理（空室一覧、募集状況、内見予約）

### 契約系
- 入居者管理（個人情報、緊急連絡先、保証人）
- 契約管理（契約作成・更新・解約、契約条件、特約）
- 入退去管理（入居日程調整、退去立会、原状回復）

### 収支系
- 家賃管理（請求・入金確認・滞納追跡・督促）
- オーナー送金（月次精算、管理費差引、送金明細）
- 経費管理（修繕費、共用部費用、仕分け）

### 運用系
- 修繕/メンテナンス管理（依頼受付→業者手配→完了報告）
- クレーム/問い合わせ管理（対応履歴、ステータス追跡）
- 書類管理（契約書PDF、鍵預かり証、写真保管）

### レポート
- ダッシュボード（稼働率、入金率、滞納件数）
- オーナーレポート（月次収支報告書の自動生成）
- 物件収支分析

## 共通ルール

- **言語**: 日本語で作業・出力（コード内コメントも日本語可）
- **金額**: 全て税込で記載し、表示時は「（税込）」を必ず付記する
- **日付**: YYYY-MM-DD 形式
- **コミットメッセージ**: 日本語、簡潔に
- **秘密情報**: `.env`, credentials, API keyは絶対にコミットしない
- **DB変更**: 必ず `supabase/migrations/` にマイグレーションファイルを作成。直接DB操作しない

### ワークフロー

- **「リリースして」= push まで含む**: コミット → push まで一気にやる
- **記事リリース**: draft から `hp/content/{date}/` に MDX 変換して配置 → リリース後draftファイルは削除 → コミット → push（contentが唯一の正）
- **HP 開発ニュース**: 開発アップデートは `hp/` にも反映

### 文体・トーン（マーケティング・記事・SNS）

- 同じ目線で語るトーンを使う
- 上から目線・偉そうな口調はNG
- 断言できないことは断言しない
- 競合の名指し・比較・批判はしない

### Ralph Loop開発フロー（PRD駆動の自律開発）

1. **PRD作成**: `PRD.md` にタスク一覧を作成
2. **PRD深掘り**: 各タスクを実装・テスト・検証・エッジケースまで含めたサブタスクに分解
3. **Ralph Loop実行**: PRDのタスクを上から順に消化。全タスクが `[x]` + 検証PASSまで止まらない
4. **完了タスクは即座に `[x]` 更新** → コミット → 次へ

### 進捗レポート出力（必須）

長時間作業中は、ユーザーが非同期で確認できるように以下のフォーマットで出力する:

**タスク完了時:**
```
[完了] Phase X > X.X タスク名
- 実装内容: 概要
- 検証結果: テスト結果 or ビルド結果
- 確認URL: （デプロイがある場合）
- 次のタスク: X.X タスク名
```

**ブロッカー発生時:**
```
[ブロック] Phase X > X.X タスク名
- 原因: 詳細
- 必要なアクション: ユーザーが何をすればいいか
- スキップして次のタスクに進みます
```

**Phase完了時:**
```
[Phase完了] Phase X: フェーズ名
- 完了: X/X タスク
- ブロック: Xタスク（内容）
- 確認URL: （あれば）
- 次のアクション: Phase X に進む or ブロッカー解消？
```

## 記事戦略（SEO優先度）

### 優先すべき記事テーマ
1. **苦情・トラブル対応系**（水漏れ、ゴミ出し、設備故障） — 検索ボリューム大、管理会社スタッフにも直接刺さる
2. **家賃滞納・督促系**（滞納初動、督促状の書き方、保証会社の使い方） — 切実な悩み、ボリューム大
3. **自主管理大家のストレス系**（自主管理きつい、夜間対応、修繕対応遅い） — 競合が個人ブログ中心で狙いやすい
4. **管理委託の判断系**（自主管理の限界、委託費用の相場） — CVに最も近い。悩み系記事からの内部リンク先として後から書く

### 後回しでいい記事テーマ
- **市場動向・時事系**（イラン戦争、ドバイミサイル等）は月1本程度に抑える。バズ可能性はあるが「賃貸管理」のドメイン専門性を積み上げにくい

### 記事の書き方ルール
- **内部リンク必須**: 新規記事公開時に既存記事への内部リンクを本文中に2〜3本入れる。既存記事からも新記事へリンクを追加する
- **テーマ一覧**: `marketing/article/draft/themes.md` に17テーマの戦略・優先順位あり

### エゴサ引っかけ戦略（実名引用による認知獲得）

記事内で **Roomlyの顧客・パートナーになりうる人や企業の実名をポジティブに引用** し、相手がエゴサーチで記事を発見 → Roomlyを認知する導線を意識的に作る。

#### 基本ルール
- **ポジティブな文脈でのみ引用する**（批判・比較はしない）
- **引用は自然な文脈で**（「〇〇の調査によると」「〇〇が詳しい」等）。不自然なねじ込みはしない
- **出典・データの引用元として使うのが最も自然**
- **個人〜小規模の専門家を狙う。** 大企業はエゴサで反応してくれる確率が低い。名前を出されたら嬉しいと感じるサイズ感がベスト
