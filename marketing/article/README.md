# Roomly マーケティング記事

## フォルダ構成

```
articles/
├── draft/      ← 執筆中・レビュー中の記事
├── released/   ← HP に公開済みの記事
└── README.md
```

## 記事フォーマット

各記事は以下の YAML frontmatter を持ちます（Luna POS と統一）。

```yaml
---
title: "記事タイトル"
description: "SEO用の説明文（2〜3文）"
date: "YYYY-MM-DDTHH:00:00"
category: "カテゴリ名"
tags: ["タグ1", "タグ2"]
---
```

### カテゴリ一覧

| カテゴリ | 内容 |
|---------|------|
| 賃貸管理の基礎 | 業界知識・基本解説 |
| 料金・コスト | 料金体系・TCO・フリーミアム |
| 導入ガイド | セルフサービス・移行・補助金 |
| 業務効率化 | 事例・効果検証 |
| 外部連携 | API・会計ソフト・銀行連携 |

### タグ一覧

| タグ | 対象読者 |
|------|---------|
| 管理会社向け | 賃貸管理会社の経営者・担当者 |
| オーナー向け | 物件オーナー・自主管理の方 |
| 導入検討中の方向け | ソフト導入を検討している方 |

## ファイル命名規則

- **kebab-case**（英語のスラッグ）
- 例: `what-is-rental-management-software.md`, `transparent-pricing.md`

## 記事の末尾構成

1. **まとめ** — 要点を箇条書きで整理
2. **よくある質問（FAQ）** — Q&A形式
3. **Roomly CTA** — 統一フォーマットのCTAセクション

```markdown
---

物件・入居者・契約・家賃・修繕をひとつの画面で管理できる。

👉 [お問い合わせ・導入の相談はこちら](https://roomly.jp/contact)
```

## ワークフロー

1. `draft/` で記事を作成・編集
2. レビュー・ブラッシュアップ
3. HP の `/content/column/` に配置してデプロイ
4. 公開後、元ファイルを `released/` に移動してコミット
