# 記事テーマ一覧

テーマと切り口案のストック。記事化したいものに `- [x]` をつける → まとめてClaude に渡すと一気にdraftを生成する。

**使い方:**
1. 記事化したいテーマの先頭チェックボックスを `- [x]` に変える
2. 「チェックしたやつdraft化して」とClaude に伝える
3. Claude がチェック済みのテーマを全てdraft化 → draft作成後は削除

切り口は「具体（入口）→ 抽象（本質）」の順で設計する。具体と抽象はなるべく遠いところを行き来する。比喩・比較もなるべく遠い業界・領域から持ち込む。**タイトル自体に「なぜその概念が？」という引っかかりを持たせる。**

## トラブル・苦情対応系

- [ ] ### 督促状は、相手が「読む気になる状態」で届いているか
> 法的に正確な文章が、最も読まれない文章になりうる
- ダイレクトメールの「開封率」研究：封筒のデザインと文体で開封率が10倍変わる（他業界：マーケティング）
- 弁護士名義の督促状は「読む」より「怖くなって逃げる」反応を引き出しやすい（具体）
- 督促状は「送った」という事実を作るためではなく「相手が読む」ために存在する
- **具体（入口）**: 担当者が手書きで書いたメールに、滞納者が「申し訳なかった」と全額返済した
- **抽象（本質）**: 読まれない督促状は、送っていないのと同じ。「正確さ」と「届くこと」は別の問題

## 自主管理大家のストレス系

- [ ] ### 夜中に鳴らなかった電話の方が、鳴った電話よりきつい
> 自主管理の消耗は「対応した数」ではなく「待機している時間」の積み上げ
- 戦場の狙撃手研究：実際に引き金を引く時間より「いつ来るかわからない」待機状態の方が精神的消耗が大きい（他業界：軍事心理学）
- 消防士の「スタンバイ疲労」：出動回数ではなく「いつ鳴るかわからない」待機が自律神経を削る（他業界：医療）
- **具体（入口）**: 「今年は何もなかった」年の方が「2件あった」年より疲れていた大家
- **抽象（本質）**: 消耗の総量は「実際に起きたこと」ではなく「起きるかもしれないと思っていた時間」で決まる。鳴らなかった電話の重さ

- [x] ### 地図が精密になるほど、迷子が増える
> 不動産の知識を増やすほど「自分の管理が正しいか」不安になる逆説
- 地図の逆説：詳細な地図を持つほど「自分が今どこにいるか」の誤差が気になる——精度が不安を生む（他業界：認知科学）
- ダニング・クルーガー効果の谷：「少し知っている状態」が最も不安が大きい。無知は不安を生まず、熟達も不安を生まない。中途半端だけが不安を生む（他業界：認知心理学）
- **具体（入口）**: 法改正を調べた大家が「今まで全部間違えていたかもしれない」と青ざめた
- **抽象（本質）**: 知識は問題を「見えるようにする」が「解決できるようにする」わけではない。見えた問題の数だけ不安が増える、という構造

## 管理委託の判断系

- [x] ### 空港では、一番目立つ出口が一番遠い
> 「管理費の安さ」「対応の速さ」——よく使われる比較軸が、最も重要でない理由
- 空港設計の研究：視認性が高い出口ほど実際の出口から遠いことが多い——目立つものが正解とは限らない（他業界：建築・動線設計）
- 意思決定研究：「比較しやすい指標」が選ばれやすいのは、それが重要だからではなく「測れるから」（他業界：行動経済学）
- **具体（入口）**: 条件だけ見て管理会社を変えた大家が、3ヶ月後に「前の方が良かった」と思った
- **抽象（本質）**: 「測れること」と「大事なこと」は別。測れる指標が意思決定を支配するとき、測れない本質が抜け落ちる

- [ ] ### 無事故の工場ほど、事故が多い
> 「長年トラブルなし」の自主管理が、最もリスクを隠しているかもしれない
- 安全工学の逆説：事故ゼロの記録が続くと「報告しなくても大丈夫」という空気が生まれ、ヒヤリハットが報告されなくなる。見えない危険が積み上がる（他業界：安全工学）
- 正常性バイアス：「今まで大丈夫だった」が「これからも大丈夫」に変換される認知の歪み
- **具体（入口）**: 「管理は得意」と思っていた大家が初めて直面した「知らなかった手続き」
- **抽象（本質）**: 無事故の記録は、リスクが存在しないことを示さない。「報告されていない」か「まだ起きていない」かを区別できない

## 経営・収支系

- [ ] ### 満室の船が、一番沈みやすい
> 稼働率100%の物件が経営危機に陥る構造
- タイタニック設計の逆説：満員の船は「浮力の余裕」がない——フル稼働はバッファゼロ状態（他業界：海運・工学）
- 軍事のサプライチェーン：余剰なく動く補給線は、一点の破綻で全体が止まる（他業界：軍事兵站）
- **具体（入口）**: 全部屋が埋まっているのに確定申告で赤字だったオーナー
- **抽象（本質）**: 「満室」という状態は余白がゼロの状態。余白のないシステムは外乱に弱い。稼働率の高さとシステムの強さは反比例することがある

- [ ] ### 修繕を先送りにする大家の物件は、なぜか「臭い」がする
> メンテナンスの先送りが、入居者に与える無意識のシグナル
- 腐敗検知の進化生物学：人間は「管理されていない空間」を無意識に嗅ぎ分ける能力を持つ。これは危険回避の本能（他業界：進化心理学）
- 割れ窓理論：一枚の割れた窓が放置されると、建物全体の「管理されていない感」が急速に広がる（他業界：犯罪社会学）
- **具体（入口）**: 故障していないのに「なんとなく古い感じ」という理由で退去が続いた物件
- **抽象（本質）**: 物件の状態は「機能するか」だけでなく「管理されているか」という信号を発している。入居者はその信号を読んで「ここに住み続けていいか」を判断している

## 入居者・市場動向系

- [ ] ### 礼儀正しい人が、一番返さない
> 入居審査で「印象」を使うことの危険性
- 詐欺師の研究：社会的信頼を得る能力が高い人ほど、長期的に被害を大きくする。「感じがいい」は社会的能力の高さを示すが、支払い能力とは無関係（他業界：犯罪心理学）
- 採用面接の研究：面接官の「この人は優秀そう」という印象は、実際の業績予測にほとんど相関しない（他業界：組織心理学）
- **具体（入口）**: 「感じが良かったから」で審査を通した入居者が4ヶ月目に失踪した
- **抽象（本質）**: 「印象」は「その場で好感を持てるか」を測っているにすぎない。それは支払い能力でも信頼性でもない。社会的印象とリスクは無相関で、むしろ逆相関することがある

- [ ] ### Googleに名前を持たない管理会社は、存在しないのと同じ
> 評価の主体が「大家」から「入居者」に移った時代の管理会社
- 地図から消えた都市は文明から消えた——記録されなかったものは、あったとしても「なかった」に近づいていく（他業界：歴史・考古学）
- 口コミの非対称性：悪い体験は積極的に書かれ、良い体験は書かれない。沈黙は「可もなく不可もない」ではなく「存在しない」と読まれる
- **具体（入口）**: 内見前にGoogleで管理会社名を検索して、何も出てこなかったから申し込みをやめた入居者
- **抽象（本質）**: 信頼は「築くもの」から「検索されるもの」に変わった。物理的に存在しても、デジタル上に存在しない信頼は、評価の土俵に乗れない

## 優先順位の考え方

1. **トラブル・対応系と滞納系から着手**。検索ボリュームが大きく、管理会社スタッフにも直接刺さる
2. **自主管理大家系は競合が個人ブログ中心**で狙いやすい。共感型の記事で流入を取る
3. **管理委託判断系はCVに最も近い**が検索ボリュームは小さい。悩み系記事からの内部リンク先として後から書く
4. **経営・収支系は内部リンクのハブ記事になる**。トラブル系が揃った後に書いて繋ぐ
5. **市場動向・時事系は月1本程度**。バズ可能性はあるが「賃貸管理」ドメイン専門性の積み上げに直結しない
