# 記事テーマ一覧

テーマと切り口案のストック。記事化したいものに `- [x]` をつける → まとめてClaude に渡すと一気にdraftを生成する。

**使い方:**
1. 記事化したいテーマの先頭チェックボックスを `- [x]` に変える
2. 「チェックしたやつdraft化して」とClaude に伝える
3. Claude がチェック済みのテーマを全てdraft化 → チェックを `- [ ]` に戻す

切り口は「具体（入口）→ 抽象（本質）」の順で設計する。

## 経営・収支系

- [ ] ### 「満室経営」を目指すと利益が減ることがある
> 空室ゼロが理想——と思い込んでいる大家は多い。でも満室に固執すると判断を間違える
- 空室を埋めるために家賃を下げすぎた結果、満室なのにキャッシュフローがマイナスになった大家（具体シーン）
- 「稼働率」と「収益性」は別の指標。どちらを優先すべきかはフェーズで変わる（抽象）
- 航空会社が「全席埋める」ではなく「収益最大化」で座席を管理している理由（他業界）
- 1部屋空いている方が、結果的に利益が出るケースとは（問いの逆転）
- **具体（入口）**: 「満室なのに手元に残らない」と気づいた大家の月末
- **抽象（本質）**: 稼働率は見た目の指標。本当の指標は手残り

- [ ] ### 家賃を下げる前にやるべきことがある
> 空室が続くと家賃を下げたくなる。でも値下げは最終手段であって、最初の一手じゃない
- 空室が3ヶ月続いて家賃を下げた大家と、写真を撮り直しただけで埋まった大家（具体対比）
- 「家賃」以外の変数（写真・文章・設備・内見対応）を先に変えたか（正解を言わない）
- 価格競争に入ると抜け出せない構造——居酒屋業界の価格戦争と同じ（他業界）
- **具体（入口）**: 家賃を下げたのに埋まらなかった大家の判断ミス
- **抽象（本質）**: 空室の原因を「家賃」と決めつけた瞬間に、解決策の探索が止まる

- [ ] ### 地主・ボンボン大家系（相続・資産家）
> 親や祖父から土地・物件を引き継いだ層。経営意識ではなく「所有意識」で動いている
- 複数テーマ/切り口のストック（記事化候補）
- 相続で突然オーナーになった30代が初めて「空室率」を計算した日（具体の一夜）
- 30年付き合いの管理会社を変えられない「義理人情」の問題（具体対比）
- 「不労所得」のつもりが、なぜか毎月何かある——「不労」の意味が全然違う（問いの逆転）
- 地主にとって建物は副産物。土地が本体という世界観と入居者のすれ違い（視点の転換）
- 3代目が初めてExcelで収支を整理したら、黒字だと思っていた物件が修繕費で食われていた（具体の一夜）

## トラブル・苦情対応系

- [ ] ### 水漏れ連絡が来たとき、最初の30分で決まること
> 初動対応の質が、その後のクレームに発展するかどうかを左右する
- 「業者に確認します」と言ったまま折り返しが来なかった事例（具体シーン）
- 初動で「どう動くか」より「どう伝えるか」の方が重要（視点の転換）
- 医療現場のトリアージと管理会社の初動の共通点（他業界）
- **具体（入口）**: 水漏れの連絡を受けた管理スタッフの最初の対応
- **抽象（本質）**: クレームは事象ではなく「放置された感覚」から生まれる

- [ ] ### ゴミ出しトラブルは「ルール」では解決しない
> 貼り紙を増やしても、ゴミの問題は解決しない。問題の構造が違う
- ゴミ置き場に注意書きを10枚貼っても変わらなかった事例（具体シーン）
- 「ルールを知らないから破る」ではなく「守る動機がない」という構造（抽象）
- コミュニティ設計と匿名性——SNSの荒れ方と同じ構造（他業界）
- **具体（入口）**: ゴミ問題が解決しない物件のループ
- **抽象（本質）**: 貼り紙は「表示」であって「解決策」ではない

- [ ] ### 騒音クレームで「どちらの肩も持たない」が正解な理由
> 管理会社が騒音トラブルに介入するとき、中立ポジションを取り続けることの重要性
- 被害者側に寄り添いすぎた管理会社が、加害者側から逆クレームを受けた事例（具体対比）
- 「解決」ではなく「対話の場を作る」が管理会社の役割（視点の転換）
- 労使紛争における調停人の立ち位置と同じ構造（他業界）
- **具体（入口）**: 騒音クレームの対応で板挟みになった管理スタッフ
- **抽象（本質）**: 管理会社が「正義」を執行しようとした瞬間に、問題はこじれる

## 家賃滞納・督促系

- [ ] ### 滞納は「2ヶ月目」に手を打てるかどうかで決まる
> 滞納初月の対応スピードが、その後の回収率を大きく左右する
- 1ヶ月目は「様子見」、2ヶ月目に動いたら手遅れだった事例（具体シーン）
- 滞納者の心理——「1回遅れると2回目のハードルが下がる」（抽象）
- 消費者金融の延滞モデルから見る、滞納の拡大パターン（他業界）
- **具体（入口）**: 「来月まとめて払います」から始まる滞納の連鎖
- **抽象（本質）**: 滞納は事後対応ではなく、予防設計の問題

- [ ] ### 保証会社があるのに、なぜ大家は滞納で疲弊するのか
> 保証会社があれば安心——と思っているが、実態は別の話
- 保証会社に請求するまでの手続きが複雑で、大家が途中で諦めるケース（具体シーン）
- 「保証される」と「手間ゼロ」は別（問いの逆転）
- 保険の「支払い拒否」問題と同じ構造——制度はあるが使えない（他業界）
- **具体（入口）**: 保証会社があるのに疲れ果てた大家
- **抽象（本質）**: 保証会社は「損害の補填」であって「問題の解決」ではない

## 優先順位の考え方

1. **トラブル・対応系と滞納系から着手**。検索ボリュームが大きく、管理会社スタッフにも直接刺さる
2. **自主管理大家系は競合が個人ブログ中心**で狙いやすい。共感型の記事で流入を取る
3. **管理委託判断系はCVに最も近い**が検索ボリュームは小さい。悩み系記事からの内部リンク先として後から書く
4. **経営・収支系は内部リンクのハブ記事になる**。トラブル系が揃った後に書いて繋ぐ
5. **市場動向・時事系は月1本程度**。バズ可能性はあるが「賃貸管理」ドメイン専門性の積み上げに直結しない
6. **地主・ボンボン系は「SaaS訴求しない」前提で書ける**。資産家の世界観・判断軸を描く読み物として面白い
