# 記事テーマ一覧

テーマと切り口案のストック。記事化したいものに `- [x]` をつける → まとめてClaude に渡すと一気にdraftを生成する。

**使い方:**
1. 記事化したいテーマの先頭チェックボックスを `- [x]` に変える
2. 「チェックしたやつdraft化して」とClaude に伝える
3. Claude がチェック済みのテーマを全てdraft化 → draft作成後は削除

切り口は「具体（入口）→ 抽象（本質）」の順で設計する。具体と抽象はなるべく遠いところを行き来する。比喩・比較もなるべく遠い業界・領域から持ち込む。**タイトル自体に「なぜその概念が？」という引っかかりを持たせる。**

## トラブル・苦情対応系

- [ ] ### 督促状は、相手が「読む気になる状態」で届いているか
> 法的に正確な文章が、最も読まれない文章になりうる
- ダイレクトメールの「開封率」研究：封筒のデザインと文体で開封率が10倍変わる（他業界：マーケティング）
- 弁護士名義の督促状は「読む」より「怖くなって逃げる」反応を引き出しやすい（具体）
- 督促状は「送った」という事実を作るためではなく「相手が読む」ために存在する
- **具体（入口）**: 担当者が手書きで書いたメールに、滞納者が「申し訳なかった」と全額返済した
- **抽象（本質）**: 読まれない督促状は、送っていないのと同じ。「正確さ」と「届くこと」は別の問題

## 自主管理大家のストレス系

- [ ] ### 夜中に鳴らなかった電話の方が、鳴った電話よりきつい
> 自主管理の消耗は「対応した数」ではなく「待機している時間」の積み上げ
- 戦場の狙撃手研究：実際に引き金を引く時間より「いつ来るかわからない」待機状態の方が精神的消耗が大きい（他業界：軍事心理学）
- 消防士の「スタンバイ疲労」：出動回数ではなく「いつ鳴るかわからない」待機が自律神経を削る（他業界：医療）
- **具体（入口）**: 「今年は何もなかった」年の方が「2件あった」年より疲れていた大家
- **抽象（本質）**: 消耗の総量は「実際に起きたこと」ではなく「起きるかもしれないと思っていた時間」で決まる。鳴らなかった電話の重さ

## 経営・収支系

- [ ] ### 満室の船が、一番沈みやすい
> 稼働率100%の物件が経営危機に陥る構造
- タイタニック設計の逆説：満員の船は「浮力の余裕」がない——フル稼働はバッファゼロ状態（他業界：海運・工学）
- 軍事のサプライチェーン：余剰なく動く補給線は、一点の破綻で全体が止まる（他業界：軍事兵站）
- **具体（入口）**: 全部屋が埋まっているのに確定申告で赤字だったオーナー
- **抽象（本質）**: 「満室」という状態は余白がゼロの状態。余白のないシステムは外乱に弱い。稼働率の高さとシステムの強さは反比例することがある

## 入退去・契約系

- [ ] ### 鍵を返した日に、入居者は初めて「住んでいた」ことに気づく
> 退去時の原状回復トラブルは「退去の日」に始まるのではなく「入居の日」に始まっている
- 所有と喪失の心理学：人間はものを失う瞬間に、それを持っていたことの価値を初めて認識する（他業界：行動経済学・保有効果）
- 退去立会いで揉める項目の多くは、入居時の写真・チェックリストがあれば防げた
- **具体（入口）**: 退去立会いで「これは最初からありました」と言い合いになった大家と入居者
- **抽象（本質）**: 退去トラブルは「退去の出来事」ではなく「入居時に何を記録しなかったか」の結果。問題は発生した時点ではなく、記録されなかった時点に遡る

- [ ] ### 契約書を読む人は、トラブルを起こさない——読まない人がトラブルを起こす
> 契約書の「読まれ方」が告げている、もっと根本的な問題
- 同意書の研究：医療のインフォームド・コンセントで、説明を読まない患者ほど術後トラブルが多い（他業界：医療倫理）
- 読まれないことを前提に設計された契約書は、トラブル防止機能を果たせない
- **具体（入口）**: 「契約書に書いてありますよ」が通じなかった入居者
- **抽象（本質）**: 契約書は「同意を取る装置」ではなく「読まれない文書」として機能している。読まれない文書が法的根拠になる、という制度の構造的な矛盾

## 管理会社の組織・人材系

- [ ] ### 管理会社の「いい担当者」は、なぜ辞めるのか
> 属人化した対応品質が、担当者の退職で一気に崩壊する構造
- 燃え尽き症候群の研究：共感能力が高い人ほど燃え尽きやすい。対人サービス業で最も優秀な人が最も早く離脱する逆説（他業界：医療・介護）
- 「あの人がいるから」で成り立っていた関係が、退職で一夜にして消える
- **具体（入口）**: 入居者から信頼されていた担当者が退職した翌月、クレームが3倍になった管理会社
- **抽象（本質）**: 属人的な「良さ」は、組織の資産ではなく個人の資産。個人が去ると一緒に消える。組織に残るのは仕組みだけ

- [ ] ### 引き継ぎが完璧だった管理会社の、引き継ぎに書かれていなかったこと
> マニュアルに書けない情報が、実は管理品質の核心にある
- 暗黙知の研究：熟練職人の技術は言語化できない部分が本質（他業界：認知科学・野中郁次郎のSECIモデル）
- 「◯◯さんは夜の電話を嫌がる」「△△号室は排水が弱いから注意」——こういう情報はマニュアルに載らない
- **具体（入口）**: 完璧な引き継ぎ書を作ったのに、後任が3ヶ月で入居者の信頼を失った
- **抽象（本質）**: 管理の質は「言語化できる情報」と「言語化できない情報」の両方で成り立っている。引き継げるのは前者だけ

## 優先順位の考え方

1. **トラブル・対応系と滞納系から着手**。検索ボリュームが大きく、管理会社スタッフにも直接刺さる
2. **自主管理大家系は競合が個人ブログ中心**で狙いやすい。共感型の記事で流入を取る
3. **管理委託判断系はCVに最も近い**が検索ボリュームは小さい。悩み系記事からの内部リンク先として後から書く
4. **経営・収支系は内部リンクのハブ記事になる**。トラブル系が揃った後に書いて繋ぐ
5. **市場動向・時事系は月1本程度**。バズ可能性はあるが「賃貸管理」ドメイン専門性の積み上げに直結しない
